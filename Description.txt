## Architecture
The app should be an static web app that uses html css and javascript. 
The architecture should be SOLID and split in multiple smaller files.
The app should be run on an github pages website so there should be a github actions file for deployment.
The app should be responsive because it will only be used on mobile

## Main menu landing page
The app should have a menu to start with. This menu should show 2 options: a select a random player option and a track life option.
Selecting 1 option should take you to the correct screen

## Track life screen
The track life screen should show 2 players by default. 
Every player should be able to add + or substract - 1 or 5 from their total and also have an option to change their color to one of the following: red,blue,green,purple,black,white,yellow,orange.

## random player
players should be able to lay a finger on the screen and after atleast 2 players have a finger on the screen and after a small delay the screen should only show 1 finger that would become the start player.
